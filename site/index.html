WinRemap

A per-application key remapper for Windows, written in Rust. Remap keys only in the apps you choose — declared in one simple TOML file.

Download installer Portable exe GitHub

Free & open source (MIT) · No network access · Never logs keystrokes · all releases

Install

  1. Download winremap-setup.exe and run it. No administrator rights are needed — WinRemap installs per-user.
  2. Choose whether WinRemap should start automatically when you sign in (recommended for daily use; you can change this later).
  3. Finish the wizard. WinRemap starts and a keyboard icon appears in the task tray.

The installer also creates a starter config at %APPDATA%\winremap\config.toml — only if you don't already have one. Your existing config is never touched, and uninstalling keeps it.

"Windows protected your PC" (SmartScreen)? WinRemap binaries are not code-signed (signing certificates are costly for a free tool), so Windows may show this warning for a new download. Click More info → Run anyway. You can — and should — verify the download instead of trusting it blindly: every release ships with a SHA256SUMS file and a GitHub build-provenance attestation proving it was built from this repository's public source. See SECURITY.md for the two commands.

Portable use (no installer)

Prefer a single file? Download winremap.exe and run it from anywhere. It reads the same %APPDATA%\winremap\config.toml, or any file you pass with --config path\to\file.toml. There are no other files or registry entries.

Quick start

WinRemap lives in the task tray. The tray menu has everything:

  • Enabled — toggle all remapping on/off without quitting
  • Settings — see the config in effect right now (below), and hand config.toml to your text editor
  • Reload config — apply config changes instantly (no restart, no lost keystrokes)
  • Show log — watch what WinRemap does with each keystroke, live
  • Quit

Open the config and describe what you want, per application:

# Ctrl+H sends a plain Backspace, but only inside Notepad
[[keymap]]
name = "notepad"
application = ["notepad.exe"]

[keymap.remap]
"C-h" = "Back"

Save, choose Reload config from the tray, and press Ctrl+H in Notepad — it now deletes one character, while every other app still sees a normal Ctrl+H.

Which name goes in application? Pick Show log from the tray and switch to the target window: the log shows each foreground app's full path, the exact application value to use, and which of your keymaps would apply.

Ready-made configs to copy from: minimal.toml (the example above), emacs.toml (fakeymacs-style Emacs bindings), and suganuma.toml (a full personal setup with exclusion lists, macros, and prefix keys).

Configuration

The config is a list of [[keymap]] blocks. Each names the applications it applies to and the key rules for them.

# Applies everywhere except the listed apps
[[keymap]]
name = "global-emacs"
application = ["*"]
exclude = ["windowsterminal.exe", "mintty.exe"]

[keymap.remap]
"C-p" = "Up"

# Applies only to these two apps — app rules always win over "*" rules
[[keymap]]
application = ["phpstorm64.exe", "idea64.exe"]

[keymap.remap]
"C-h" = "Back"

A rule's left side is what you press, its right side is what the app receives. "C-h" = "Back" matches that exact chord and replaces the modifiers too; "CapsLock" = "LCtrl" swaps the key whatever else is held. Two-stroke prefixes ("A-x u") and macros (["C-Right", "C-Left", "C-S-Right"]) go in the same place.

Config errors are reported all at once, with line numbers. If a reload fails, the previous working config stays active — a typo never disables your keyboard setup.

Macro recording (optional)

Name the keys and WinRemap can record what you do and play it back. Nothing happens until you name them.

[macro]
record_start = "S-F10"  # press to start recording
# record_stop = "S-F11" # omit it and the start key stops it too
record_play  = "F10"    # press to replay

Press Shift+F10, do the work, press it again, then press F10 as often as you like. While a recording runs, a banner sits at the bottom of whichever display holds the app you are typing into:

Recording macro  3/20   notepad.exe in progress   —   S-F10 to stop, F10 to replay

It follows you as you switch apps and displays, names the app receiving the keystrokes, and repeats the keys that end and replay the recording — so a recording never runs unnoticed, and you never have to open the config file to find out how to stop it. Replay shows the same banner with the commands it is sending.

Worth knowing before you rely on it:

  • 20 commands per recording. A command is one chord: C-a is one, the two strokes of A-x h are two. The 21st ends the recording and says so; nothing is dropped quietly.
  • Nothing is written to disk. The recording lives in memory and is gone when WinRemap exits, and it is never saved to your config file. To keep a sequence, write it as a macro rule instead.
  • What is recorded is what WinRemap emitted, not the keys you pressed. With "C-h" = "Back" in effect, the recording holds Back.
  • Order, not timing. Each command is tapped once, in order. How long you held a key, and holding one key while pressing another, are not reproduced — Emacs keyboard macros behave the same way.
  • One recording is kept; recording again replaces it. The replay key does nothing while a replay is running, and an in-progress recording is dropped if you disable WinRemap or reload the config, since the key that would end it comes from that file.

delay_ms paces a replay exactly as it paces a config macro.

IME status indicator (optional)

Independent of remapping, WinRemap can flash a translucent 「あ」 panel at the center of the active window the moment the IME turns on — or when you focus a window whose IME is on — so you always know your input mode before typing. Display only: WinRemap never switches the IME.

[ime_indicator]
enabled = true                # default: false
# trigger_keys = ["C-Space"]  # if you toggle the IME with Ctrl+Space
# duration_ms = 800           # 100–5000
# size = 96                   # 32–256
# opacity = 200               # 0–255
# show_app_name = true        # exe name under the glyph

Standard IME keys (Henkan/Muhenkan, Zenkaku/Hankaku, Kana, IME On/Off) are detected out of the box; add trigger_keys if you use a custom toggle like the Windows 11 IME's Ctrl+Space option. The panel never takes focus or input, and a problem in the indicator never affects remapping.

FAQ & troubleshooting

Remapping doesn't work in one specific window

If that window runs as administrator, Windows blocks input from non-elevated programs (UIPI). Run WinRemap elevated too if you really need remapping there. Games with anti-cheat and some virtualization software also ignore injected input.

How do I check what is actually in effect?

Right-click the tray icon and pick Settings. It shows the config WinRemap is resolving against right now — every keymap, its target apps and exclusions, and its rules, each with the comment you wrote beside it in the file. Where the same input is bound in more than one keymap, a column names the others, since only one of them can win.

The window is read-only in this release. Edit the file and press Reload config; the display follows. The file's modification time and the load time sit side by side, so a config you saved but did not reload is obvious.

My rule doesn't match — how do I debug it?

Right-click the tray icon and pick Show log. It shows the exact application value for each foreground window and logs every key decision (pass-through / remap / macro / prefix) at key-name level, as you type. Prefer a terminal? Start WinRemap with winremap.exe --debug and the same lines go there. Debug output never includes typed text and is never written to disk.

Can I use it together with Keyhac / AutoHotkey?

Not recommended for the same keys: stacked low-level keyboard hooks have undefined ordering, so behavior becomes unpredictable. Disable one while testing the other.

Where did the console window go?

WinRemap no longer opens one: launching it from Explorer, the Start menu or at sign-in is silent, and anything you must see (a startup error, a failed reload) appears as a dialog. Start it from a terminal and it still prints there, though the prompt returns immediately so output arrives interleaved with it. For a readable running log, use Show log in the tray.

How do I turn autostart on or off after installing?

Press Win+R, run shell:startup, and add or remove the WinRemap shortcut there. (Re-running the installer also lets you re-choose the option.)

A macro types too fast for the target app

Set delay_ms = 8 under [macro] (0–15) — some apps sample input asynchronously and drop burst-injected strokes.

Where does WinRemap store things?

Program: %LOCALAPPDATA%\Programs\WinRemap (installer) or wherever you put the portable exe. Config: %APPDATA%\winremap\config.toml. That's all — no services, no drivers, no registry settings.

How do I verify my download is genuine?

Compare the SHA-256 hash against SHA256SUMS from the release, and check the build-provenance attestation with the GitHub CLI:

(Get-FileHash .\winremap-setup.exe -Algorithm SHA256).Hash.ToLower()
gh attestation verify .\winremap-setup.exe --repo DaikiSuganuma/winremap

Details in SECURITY.md. Binaries from anywhere other than GitHub Releases are unofficial.

Security & scope

  • WinRemap never logs or stores keystrokes — enforced as repository policy, not just good intentions.
  • No network code at all: no telemetry, no auto-update.
  • The hook callback is pure Rust with no heap allocation, locking, or I/O, which keeps worst-case latency low and avoids the hook timeouts that plague script-based remappers.
  • Out of scope by design: IME control (the indicator only displays state), clipboard tools, launchers, macOS/Linux (use xremap on Linux).
