# ADR 0034: egui の未確定文字表示は下線ありにする（legacy_visuals = false）

- ステータス: accepted
- 日付: 2026-07-21
- 作成: Claude Code（AI モデル: claude-opus-4-8）／レビュー・承認: オーナー（2026-07-21）

## 文脈

ADR 0030 が「設定 GUI の設計前に潰すべきリスク」として挙げた 3 点のうち、最後まで残っていた
「日本語 IME での入力」を `examples/egui_ime_probe.rs` で検証した（受け入れチェックリスト
B-1〜B-8）。

入力そのものは問題なく動いた（未確定文字は欄内に出る、変換候補ウィンドウはキャレット付近に
出る、`Preedit` → `Commit` が流れる、Shift + カーソルキーで文節を移動できる、WinRemap 常駐中
でも取りこぼしがない）。ただし **未確定文字に下線が付かなかった**。

原因は egui の既定値だった。`egui::style::Visuals::ime_composition.legacy_visuals` は
**Windows でのみ既定 `true`** であり、`true` のときは未確定文字を選択範囲と同じ見た目で描画し、
下線を引かない。egui のドキュメントコメントによれば、Windows だけ既定を変えている理由は
**winit が韓国語 IME でカーソル位置を誤って報告する不具合**であり、日本語入力のためではない。

`false` にすると本来の描画になる。

- 未確定文字に下線が付く（`active_underline_stroke` / `inactive_underline_stroke`）
- 変換対象の文節とそれ以外の文節が別の線で区別される
- 未確定文字列の中でのカーソル位置が表示される

## 決定

**設定 GUI のテキスト入力欄では `Visuals::ime_composition.legacy_visuals = false` を設定する。**

- 適用単位は GUI 全体のスタイル。個々のウィジェットで切り替えない
- `examples/egui_ime_probe.rs` は既定を `false` とし、チェックボックスで `true` と見比べられる
  ままにしておく（将来 egui の既定が変わったときの回帰確認用）

あわせて、ADR 0030 の 3 リスクはこれで全て解消とする。

| リスク | 結論 | 根拠 |
|---|---|---|
| 日本語フォントの読込方式 | システムフォントを実行時に読み込む | A-11（`src/log_window.rs` で実装済み） |
| フック稼働中の GUI 開閉 | `eframe` で足りる。GUI は専用スレッド、閉じるときは隠す（[ADR 0032](0032-log-window-hide-instead-of-close.md)） | A-12・A-13・A-17 |
| 日本語 IME での入力 | 問題なし。未確定表示のみ本 ADR で調整 | B-1〜B-8 |

参照:

- `egui::style::ImeComposition`（`legacy_visuals` の既定値と理由）:
  <https://docs.rs/egui/0.35.0/egui/style/struct.ImeComposition.html>

## 理由

- **日本語入力の見やすさが直接効く**。WinRemap の UI 言語は日本語と英語であり、`application`
  名やキーマップ名の入力で日本語を打つ場面がある。未確定文字が確定文字と見分けられないのは
  実用上の不便であり、オーナーも見比べたうえで下線ありを選んだ
- **Windows 既定が `true` である理由が当てはまらない**。韓国語 IME のカーソル位置バグへの
  回避策であり、日本語入力での既知の問題ではない
- **1 行の設定で戻せる**。将来 egui 側で新描画に問題が見つかっても、`true` に戻すだけで済む

## 却下した代替案

- **既定（`true`）のままにする**: 実装は何もしなくてよいが、未確定文字と確定文字が見分けられず、
  文節の移動も目視で確認できない。オーナーの比較で明確に劣ると判断された
- **設定ファイルで切り替え可能にする**: 選択肢を増やすだけで、どちらが良いかは明らかである。
  「単純さ > 機能の豊富さ」の優先順位に反する

## 残るリスク

韓国語 IME を使う利用者では、未確定文字列内のカーソル位置が正しく表示されない可能性がある
（winit 側の不具合）。日本語・英語 UI の本ツールで韓国語を入力する場面は限定的であり、
影響は表示のみで入力自体は成立する。問題が報告されたら本 ADR を新しい ADR で覆す。
