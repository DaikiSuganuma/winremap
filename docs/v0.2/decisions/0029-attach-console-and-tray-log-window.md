# ADR 0029: 起動時コンソールの廃止と、AttachConsole + トレイ「ログ表示」ウィンドウによるログ出力

- ステータス: 承認（オーナー決定 2026-07-20）
- 日付: 2026-07-20
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

v0.1 は console subsystem でビルドされているため、エクスプローラーやスタートメニューから起動するとログ出力用のコンソールウィンドウが表示される。常駐ツールとして起動時に余計なウィンドウを出さないことが v0.2 の要件となった（[v0.2 開発計画](../01_development-plan.md) Phase A）。

`#![windows_subsystem = "windows"]` に変更すると stdout / stderr は既定でどこにも接続されないため、(1) ターミナル利用時の `--debug` / `--help` の出力先、(2) 一般利用者がログを見る手段、の 2 つを決める必要がある。

## 決定

1. **ターミナルから起動されたときだけ、そのターミナルに出力する** — `AttachConsole(ATTACH_PARENT_PROCESS)` で親プロセスのコンソールに接続する。エクスプローラー等からの起動では接続先がないため無音になる
2. **タスクトレイの右クリックメニューに「ログ表示」を追加する** — 選択すると専用ウィンドウが開き、開いている間はデバッグモードのログをそのウィンドウへ出力する。ログイベントはフックコールバックから既存のロックフリーキュー（ADR 0016 の機構）で受け渡し、フックコールバック内に処理を追加しない
3. 無音起動時の設定エラー等は、コンソールの代わりにメッセージボックスまたはトレイ通知で利用者に見せる（実装時に詳細設計）

参照（公式）:

- windows_subsystem 属性: https://doc.rust-lang.org/reference/runtime.html#the-windows_subsystem-attribute
- AttachConsole: https://learn.microsoft.com/en-us/windows/console/attachconsole

## 理由

- 開発者・上級者のワークフロー（ターミナルから `--debug` で起動して挙動を追う）は v0.1 と同等に保たれる
- 一般利用者はターミナルを開かなくても、トレイから 2 クリックでログを確認できる。「困ったらログ表示を開いてスクリーンショットを送る」というサポート導線が作れる
- ログの受け皿が両方ともオンメモリ表示であり、ディスクに永続化しないため、不変条件 6（キーロガー化の禁止。デバッグログはキー名レベルまで・既定 OFF）と整合する

## 却下した代替案

- **`--debug` 時のみ `AllocConsole`**: トレイの「ログ表示」ウィンドウが同じ役割をより良い UX で果たすため不要。コンソールはコピーやスクロールの操作性も劣る
- **ログファイル出力**: `--debug` ログはキー名レベルの情報を含むため、ディスク永続化は不変条件 6（キーロガー化の禁止）に抵触するリスクがある。採用しない
- **`OutputDebugStringW`**: DebugView 等の外部ツールが必要で、実質開発者専用。一般利用者のログ確認手段にならない
- **Windows イベントログ**: 常駐ツールのキー名レベルのログを OS のイベントログに残すのは不適切かつ過剰。ソース登録の運用も煩雑

## 影響・補足

- **unsafe 隔離リストへの例外追加（オーナー承認済み 2026-07-20）**: `AttachConsole` は Windows API のため呼び出しに `unsafe` が必要だが、その役割（起動時のコンソール接続）は不変条件 3 が許可する 4 か所（hook.rs / sender.rs / window.rs / ime_indicator/）のいずれにも属さない。ターミナル出力はデバッグ時に有用であるため機能を維持し、**専用ファイルを unsafe 許可リストに追加する方針**をオーナーが承認した。実装着手時に、追加するファイル名（`console.rs` 等）を確定させて AGENTS.md 不変条件 3 とブリーフ §5-3 を改訂する ADR を別途起こす（ADR 0009 と同じ扱い。安全にラップされたクレートで代替できる場合は依存追加のみで済ませる案も併せて検討する）
- cmd / PowerShell から起動した場合、プロンプト表示と出力が混ざる（親コンソール接続の既知の仕様）。ヘルプサイトの Troubleshooting に記載する
