# ADR 0042: ウィンドウ内のアイコンは白で焼き、描画時に着色する

- ステータス: accepted
- 日付: 2026-07-21
- 作成: Claude Code（AI モデル: claude-opus-4-8）／レビュー・承認: オーナー（2026-07-21）

## 文脈

オーナーの要望: 設定ウィンドウのセクション見出し（対象アプリ・除外アプリ・リマップ規則）の
前にもアイコンを表示したい。

[ADR 0040](0040-menu-icons-rasterized-at-build-time.md) でトレイメニュー用に
Bootstrap Icons をビルド時ラスタライズする仕組みは既にある。しかし、そのまま流用すると
2 点合わない。

1. **色**: メニュー用は暗いグレー（`#343a40`）で焼いている。Win32 のメニューは完成した
   ピクセルを受け取るだけで着色できないためである。ウィンドウ側は egui が描くので、
   ダークテーマでは暗いアイコンが背景に沈む
2. **解像度**: メニュー用は 16×16 固定。ウィンドウは HiDPI ディスプレイで拡大描画されるため、
   16px のまま引き伸ばすとぼやける

## 決定

**ウィンドウ用は別セットとして、白で 32px に焼き、egui の `tint` で描画時に着色する。**

- `build.rs` の出力を用途別に分ける（`menu-*.rgba` = 16px・暗いグレー、`ui-*.rgba` =
  32px・白）。同じアイコンを両方で使う日が来ても衝突しない
- 描画時の色は `ui.visuals().text_color()`。**明暗テーマの切り替えに自動追従する**
- 論理 16pt で描き、素材は 32px。等倍表示では縮小がかかるが、拡大でぼやけるより良い
- テクスチャは `Context` に 1 度だけアップロードし、`data` にハンドルを保持する。
  毎フレームのアップロードは避ける
- アイコンは用途名（`Icon::Apps` / `Icon::Rules` …）で参照する。Bootstrap のアイコン名は
  `icons.rs` の中だけに閉じ込め、絵柄の差し替えで呼び出し側を触らずに済むようにする

## 理由

- **白＋着色は素材が 1 枚で済む。** 明暗 2 種を焼き分けると、テーマ判定と 2 倍の
  ピクセルが必要になる（ADR 0040 の「残るリスク」で挙げた手法をウィンドウ側では
  採らずに済む、ということでもある）
- **ビルド時ラスタライズという ADR 0040 の骨子は変えていない。** 実行ファイルに
  resvg は入らず、増えるのは 32×32×4 = 4KB × 6 個のピクセルのみ
- 実測: リリース exe の増分は誤差の範囲（8.4MB → 8.1MB。LTO の揺れの方が大きい）

## 却下した代替案

- **メニュー用の 16px 暗色をそのまま使う**: 素材は増えないが、ダークテーマで沈み、
  HiDPI でぼやける。アイコンを付ける目的（見出しの識別）を果たさない
- **明暗 2 種を焼き分ける**: テーマ判定が要る。着色 1 回で済む話に手数が見合わない
- **`egui_extras` の svg 機能でランタイム描画**: 常に最適な解像度で描けるが、
  resvg + usvg + tiny-skia が実行ファイルに入る。ADR 0040 で却下した理由がそのまま当てはまる
- **絵文字で代用**（`📋` `⚡` 等）: 依存ゼロだが、字形がフォント任せで揃わず、
  カラー絵文字はテーマに追従しない
