# ADR 0041: ログのセッション境界は Win32 のローカル時刻で記録する

- ステータス: accepted
- 日付: 2026-07-21
- 作成: Claude Code（AI モデル: claude-opus-4-8）／レビュー・承認: オーナー（2026-07-21）

## 文脈

オーナーの要望: ログ表示の先頭に起動日時とバージョンを出し、終了時にはコンソールへ
終了日時とバージョンを出したい。

ログウィンドウは起動から何時間も経ってから開かれる。開いた時点のバッファは空なので、
「このセッションはいつ始まったのか」がログだけからは分からない。バージョンも同様で、
利用者が貼り付けたログを読む側にとっては、どのビルドの挙動なのかが最初に必要になる。

問題は時刻の取得方法である。Rust の標準ライブラリには**ローカル時刻が無い**。
`std::time::SystemTime` は UNIX エポックからの経過時間しか返さず、タイムゾーンの
知識を持たない（[std::time のドキュメント](https://doc.rust-lang.org/std/time/struct.SystemTime.html)）。

## 決定

**`src/clock.rs` を追加し、Win32 の `GetLocalTime` でローカル時刻を取得する。
このファイルを unsafe 許可リストに加える（AGENTS.md 不変条件 3）。**

- 書式は `YYYY-MM-DD HH:MM:SS` 固定。ロケール依存の書式にはしない
- 起動時にバナー行を作り、コンソールへ出すと同時に `gui::log::set_session_start` へ渡す。
  ログウィンドウは開くたびにこれを先頭に置く（バッファは閉じるたびに破棄されるため、
  バナーはバッファの外に保持する）
- 終了時は `run()` の末尾で `gui::log::emit` する。コンソールに出て、
  ログウィンドウが開いていればそこにも残る
- `windows` クレートの機能 `Win32_System_SystemInformation` を追加する

## 理由

- **`chrono` / `time` クレートを足すより軽い。** 必要なのは 1 か所・1 行の書式化だけで、
  そのために依存を 1 つ増やすのは「単純さ」の優先順位に反する
- **`GetLocalTime` は失敗しない。** 引数は出力用の `SYSTEMTIME` 1 つで、戻り値も無い。
  unsafe としては最も浅い部類であり、隔離のコストが小さい
- **書式を固定する理由**: ログは Issue に貼られる。並べ替え可能で、月日の順序が
  読み手の地域に依存しない形式でなければならない
- フックの経路上では動かない（起動時と終了時の各 1 回のみ）

## 却下した代替案

- **`chrono` を追加する**: タイムゾーンデータベースを含む大きな依存を、日時 1 行のために
  引き込むことになる。配布サイズと監査対象が増える
- **UTC で記録する**（`SystemTime` のみで完結）: 依存も unsafe も増えないが、
  利用者が「さっきキーを押した時刻」と突き合わせられない。ログの用途を満たさない
- **`notify.rs` に時刻取得を同居させる**（既に許可リスト上にある）: 新規ファイルも
  ADR も不要になるが、`notify.rs` は「利用者への出力経路」の責務であり、
  時計を混ぜると次に時刻が要る場所から `notify` を呼ぶことになる
- **起動からの経過時間（`Instant`）を出す**: 依存ゼロだが、絶対時刻でないと
  他のログ（イベントビューア等）と突き合わせられない
