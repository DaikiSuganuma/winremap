## 実施上の前提（重要）

**確認は WinRemap を 1 つだけ動かした状態で行う。** 既にインストール版が常駐している場合は先に終了させること。二重フックは挙動不定になり、結果が信用できない（ブリーフ §9・README Limitations）。

## Phase B — egui 日本語 IME プローブ（ADR 0030 のリスク 3）

> `cargo run --example egui_ime_probe` で実行する。**WinRemap を常駐させたまま**実施すること（フックとの同居も同時に見るため）。フックもトレイも持たないプローブなので、二重フックにはならない。

| # | 手順 | 期待する結果 | 結果 |
|---|---|---|---|
| B-1 | 単一行の欄（1）に IME をオンにして日本語を入力 | 未確定文字が欄の中に表示され、確定すると欄に入る | OK（下線なし。原因判明・下記 B-8） |
| B-2 | 変換候補ウィンドウの位置を見る | キャレット付近に出る（画面左上や欄と無関係な位置ではない） | OK |
| B-3 | 複数行・等幅の欄（2）に日本語を複数行入力 | 同上。改行しても候補位置が追従する | OK |
| B-4 | 表の行内編集（3）の各欄に日本語を入力 | 同上。欄を移動しても未確定文字が別の欄に混ざらない | OK |
| B-5 | 変換中に Esc・変換・無変換を押す | 候補操作ができ、欄からフォーカスが外れない | OK。Shift+カーソルキーでの文節移動も可 |
| B-6 | 画面下部の「直近の IME イベント」を見る | `Preedit` → `Commit` が並ぶ（IME 経路が生きている証拠） | OK |
| B-7 | プローブに入力している間、WinRemap のリマップを試す | リマップが従来どおり動作し、入力の取りこぼしがない | OK |
| B-8 | プローブの「旧来の未確定表示」チェックを**外して**日本語を入力 | 未確定文字に**下線が付き**、変換対象の文節が他の文節と区別できる | OK。下線ありを採用（[ADR 0034](decisions/0034-ime-preedit-underline-visuals.md)）|

判定: B-1〜B-6 が通れば設定 GUI にテキスト入力欄を置いてよい。落ちた場合は入力欄を避ける設計（キーキャプチャ・選択リスト中心）へ切り替えるか、フレームワーク選定（ADR 0030）を見直す。

**2026-07-21 実施結果**: B-1〜B-7 は期待どおり。B-1 で下線が出なかったのは egui の既定値によるもので、`Visuals::ime_composition.legacy_visuals` が **Windows でのみ既定 `true`**（未確定文字を選択範囲と同じ見た目で描き、下線を引かない）ためと判明した。`false` にすると下線と文節区別が有効になる。プローブに切り替えチェックボックスを追加したので B-8 で確認する。
