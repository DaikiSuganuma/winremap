# ADR 0010: アイコンは exe リソースとして埋め込み、トレイはリソースから読む

- ステータス: 承認
- 日付: 2026-07-18
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

オーナーがデザインしたアイコン（`assets/` — .ico 2 種（有効/無効、各 16/24/32/48/256 内包）、PNG 各サイズ、SVG マスター）を、コード生成の暫定アイコン（[07_icon-design-brief.md](../04_icon-design-brief.md)）と置き換える。単一バイナリ配布を維持したまま組み込む方式の選定が必要。

## 決定

- **`build.rs` + `winresource` クレート（build-dependency、MIT）で 2 つの .ico を Windows リソース（ordinal 1 = 有効、2 = 無効）として exe に埋め込む**
- `tray.rs` は `tray_icon::Icon::from_resource(ordinal, None)` で読み込む。サイズ `None` によりシェルが DPI に応じて .ico 内の最適サイズを選ぶ
- ordinal 1 は Windows の慣習で exe 本体のアイコン（エクスプローラー・タスクマネージャー表示）を兼ねる

## 理由

- 実行時の画像デコード依存が不要（png クレート等を足さない）
- 1 つの仕組みでトレイアイコンと exe アイコンの両方が設定される
- .ico は複数サイズを内包するため、トレイの DPI スケーリング（16/20/24px 表示）に追従できる
- `winresource` はビルド時のみの依存で、配布バイナリに増分がない（リソース分の数十 KB のみ）

## 却下した代替案

- **png クレートで埋め込み PNG を実行時デコード**: 実行時依存が増え、exe アイコンは別途設定が必要になる
- **生 RGBA blob をリポジトリにコミットして `include_bytes!`**: 依存ゼロだがリポジトリ内が不透明なバイナリになり、レビュー不能。DPI 別サイズも持てない
- **コード生成アイコンの継続**: オーナーデザインが用意された時点で役割終了（暫定案として ADR 0007 に記録済み）

## 備考

- `winresource` は MSVC ツールチェーンの `rc.exe` を使用する。ローカル（MSVC）と CI（windows-latest）の双方に存在するため追加セットアップ不要
- アセットの正本は `assets/svg/`。変更時は PNG / .ico を再生成してコミットする
