# ADR 0021: IME インジケーターのトリガーキーを設定可能にする

- ステータス: 承認（オーナー検証で発覚した問題への対処）
- 日付: 2026-07-19
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

Phase I3 のオーナー動作確認で「パネルが一切表示されない」と報告された。デバッグログの分析により、オーナーの IME 切替は **Ctrl+Space**（Windows 11 IME の「Ctrl + Space を使用して IME のオン/オフを切り替える」オプション）であり、ハードコードしたトグル候補 VK セット（変換・無変換・かな・半角/全角・IME On/Off）に含まれず、フックから indicator スレッドへの通知が一度も飛んでいなかったことが判明した。

IME 切替キーはユーザーが IME 設定で自由に割り当てられるため、固定 VK セットだけでは原理的に取りこぼす。

## 決定

1. `[ime_indicator]` に `trigger_keys`（キー記法の文字列配列、例 `["C-Space"]`）を追加する
2. 既存のキー記法パーサ（`parse_key_combo`）を再利用し、不正な記法・修飾キー単体は行番号付き設定エラーにする（修飾キーはフックがコマンド状態管理に消費するため通知点に到達せず、指定しても永遠に発火しない）
3. 設定されたトリガーは**修飾キー込みの完全一致**で判定する（`C-Space` は素の Space では発火しない）。組み込み VK 候補は従来どおり修飾キー無視で判定し、`trigger_keys` は**追加**（置き換えではない）とする
4. フック内の判定は arc-swap 済みテーブルの **借用比較のみ**（小さな Vec の `contains`）とし、クローン・アロケーションを行わない（不変条件 2 の維持。`IndicatorSettings` が `Vec` を持ち `Copy` でなくなったため、フック経路からスナップショット取得関数を呼ばないことをコメントで明示）

## 理由

- ユーザー割り当てのトグルキーは環境依存であり、設定で表現するのが唯一の一般解
- キー記法・パーサ・エラー報告の既存基盤をそのまま使えるため、設定面の追加コストが小さい
- 追加方式（組み込み候補＋ユーザー指定）なら、標準的なキー（変換/無変換等）を使う利用者は設定不要のまま

## 却下した代替案

- **全キー押下で照会を発火**: 設定不要になるが、タイピング中は常時ポスト＋照会が走る。デバウンスで実害は小さいものの、承認済み例外（「トグル候補キー検知での PostThreadMessageW」）の範囲を超え、不変条件 2 の例外を「全キー」に広げることになるため却下
- **Ctrl+Space をハードコード追加**: オーナー環境は直るが、次の環境依存キーでまた同じ問題が起きる
- **組み込み候補の置き換え方式**: `trigger_keys` を指定した瞬間に変換/無変換等が効かなくなる罠になるため、追加方式とした
