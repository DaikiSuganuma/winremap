# ADR 0012: マクロ出力（RHS のチョード配列）

- ステータス: 承認（オーナー指示による v0.2 機能の前倒し導入）
- 日付: 2026-07-18
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

fakeymacs の `self_insert_command("C-Right", "C-Left", "C-S-Right")`（単語選択）のような「1 入力 → 複数キーの順次送出」が v0.1 では表現できず、Keyhac からの移行の障害になっていた。

## 決定

RHS に文字列配列を許可し、各要素のチョードを順に 1 回ずつタップするマクロ出力とする（`"C-t" = ["C-Right", "C-Left", "C-S-Right"]`）。

- **上限 8 チョード**（`keymap::MAX_MACRO_LEN`）。送出バッチのスタックサイズをコンパイル時に確定させるための制限で、超過は設定エラー
- 全要素（修飾キーの一時解除・復元含む）を **1 回の `SendInput` バッチ**で送出する。バッチ内には実入力が割り込まないため、マクロの途中に物理キーが混ざらない
- 実行は**キー押下時に 1 回**。キーリピートでは再実行しない（押しっぱなしでマクロ連射される事故の防止）。元キーの keyup は抑止のみ
- 単キールール（修飾なし LHS）の RHS には不可（単キールールは「キー差し替え」であり意味論が異なる）

## 理由

- 送出中の割り込み防止と「フック内でヒープ確保しない」制約（AGENTS.md 不変条件 2）を、固定長スタックバッチという 1 つの仕組みで両立できる
- リピート無効は安全側の選択。必要なら都度押せばよく、暴発の被害（例: 改行連打）のほうが大きい

## 却下した代替案

- **要素ごとに `SendInput` を分割呼び出し**: 高速タイピング時に実入力がマクロ列に割り込み、選択操作などが壊れる
- **リピートでの再実行**（Keyhac 互換）: 上記の暴発リスクを取らない。要望が出たら設定オプションとして再検討
- **上限なし（Vec を都度送出）**: フック内での動的サイズ処理になり不変条件に抵触する
