# ADR 0001: Win32 API バインディングに windows-rs を採用する

- ステータス: 承認
- 日付: 2026-07-18

## 文脈

winremap は `WH_KEYBOARD_LL` フック、`SendInput`、前面ウィンドウ取得などの Win32 API を直接呼び出す。Rust から Win32 を呼ぶバインディングには主に `windows`（windows-rs）、`windows-sys`、`winapi` がある。

## 決定

**`windows` クレート（windows-rs、Microsoft 公式）を採用する**。実際に依存へ追加するのはフック層を実装する Phase 2 とし、必要な feature のみ有効化する。

## 理由

- Microsoft 公式で、Windows SDK のメタデータから生成されており API カバレッジと正確性が高い
- 継続的にメンテナンスされている
- `Result` ベースのエラーハンドリングなど Rust らしい型付けがあり、unsafe ラッパーを書きやすい
- feature 単位で必要な API のみ有効化でき、ビルド時間と依存面積を制御できる

## 却下した代替案

- **`winapi`**: 長期間メンテナンスが停滞しており（0.3 系で事実上凍結）、新しい API の追加が見込めない。ブリーフ §4 でも不採用と明記
- **`windows-sys`**: 生の extern 宣言のみで最軽量だが、型安全性の支援がなく unsafe ラッパーを全て手書きすることになる。本プロジェクトは依存最小化よりフック層の安全性・可読性を優先する（迷ったときの優先順位: 安定性 > 単純さ）。ビルド時間が問題になった場合に再検討する
