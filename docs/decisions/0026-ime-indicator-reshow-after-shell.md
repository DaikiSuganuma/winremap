# ADR 0026: シェル面経由でアプリに戻ったときはインジケーターを再表示する

- ステータス: 承認（オーナー指示。[ADR 0023](0023-ime-indicator-query-target.md) の一部を上書き）
- 日付: 2026-07-19
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

ADR 0023 はシェル面（タスクバー・デスクトップ等）のクリックで「表示・非表示・状態更新のすべてを行わない」と決めた。状態を保持するため、タスクバーをクリックして元のアプリに戻っても「対象が変わった」と判定されず、パネルは再表示されない。

実使用でオーナーから「タスクバーを経由して元のアプリに戻ったときも再表示してほしい」との要望があった。タスクバー操作を挟むと入力モードの確信が薄れるため、アプリへフォーカスが戻った時点での再通知に価値がある、という利用実感に基づく。

## 決定

- シェル面を観測したら、**直前の対象ウィンドウの記憶（last_target）だけをリセット**する。シェル面そのものに表示しない点・IME 状態の記憶（last_on）を保持する点は従来どおり
- これにより、シェル面経由で戻った先の照会は「対象ウィンドウが変わった」と判定され、IME がオンならパネルが再表示される。オフなら何も出ない（誤表示より非表示の原則は維持）
- ADR 0023 のステータスに本 ADR による一部上書きを注記する

## 理由

- 変更は 1 行（センチネルへのリセット）で、シェル面無視の原則・障害分離・状態機械の単純さを崩さない
- 「フォーカスが戻った＝入力を再開する瞬間」であり、フォアグラウンド切替トリガーの本来の目的（切替先でオンだと気づける）と整合する

## 却下した代替案

- **状態を完全保持（従来の ADR 0023 の挙動）**: 点滅抑止としては正しいが、オーナーの利用実感では「戻ったときに出ない」方が不便だった
- **シェル面クリック時にパネルを即時非表示にする**: フェードアウト中の見た目を不自然に断ち切るだけで、再表示の要望には応えない
- **シェル面も通常ウィンドウと同様に扱う（無視をやめる）**: タスクバー中央にパネルが出る元の問題（ADR 0023 の発端）が再発する
