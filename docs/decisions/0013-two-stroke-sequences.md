# ADR 0013: 2 ストロークシーケンス（プレフィックスキー）

- ステータス: 承認（オーナー指示による v0.2 機能の前倒し導入）
- 日付: 2026-07-18
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

fakeymacs の Ctl-x プレフィックス（`fc.ctl_x_prefix_key = "A-x"` で A-x → C-x 系列）に相当する 2 ストローク入力（`C-x C-s` のような）が必要。ブリーフ §3.2 の v0.2 候補で、オーナー指示により導入する。

## 決定

LHS を空白区切りの 2 チョード（`"A-x u"`、`"A-x C-s"`）とし、次の意味論で実装する。

- **プレフィックス（1 打鍵目）は修飾キー付きチョード必須**。素のキー（`"x h"`）は通常タイピングを飲み込むため設定エラー
- プレフィックス押下は抑止し、フックの thread_local に保持（pending 状態）。**タイムアウトなし**（Emacs と同じ。フック内にタイマーを持ち込まない実装上の利点もある）
- pending 中の修飾キーイベントは素通し（Alt を離してから 2 打鍵目を打てる）。次の**非修飾キー押下**が 2 打鍵目として消費される
- **未定義の 2 打鍵目は握りつぶす**（Emacs の undefined key 相当。タイプミスがアプリに入力されない）
- 2 打鍵目の解決はアプリ別 > グローバルの通常優先順位で行う（プレフィックスがグローバル定義でも、2 打鍵目はアプリ別定義が勝てる）
- 同一チョードの「通常ルール」と「プレフィックス」の重複は設定エラー（押下時にどちらか判定できないため）
- ストローク数は 2 まで。出力は単一チョード・マクロ（ADR 0012）のどちらも可
- トレイで無効化した際は pending 状態もクリアする

## 理由

- fakeymacs の実用パターン（A-x プレフィックス）を、フックの不変条件（ヒープ確保・タイマーなし）を守ったまま Cell 1 個の状態追加で実現できる
- 「未定義は握りつぶす」「タイムアウトなし」は Emacs ユーザーの既存の期待に一致する

## 却下した代替案

- **未定義の 2 打鍵目を素通しする**: プレフィックスだけ消えて 2 打鍵目が入力される中途半端な状態になり、タイプミス時の挙動が予測しにくい
- **タイムアウト付き pending**: フック外にタイマースレッドが必要になり複雑化。Emacs にもない挙動
- **任意長シーケンス（3+ ストローク）**: 状態機械が複雑になる割に fakeymacs 移行の用途では不要。必要になれば別 ADR
