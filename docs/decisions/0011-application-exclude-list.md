# ADR 0011: グローバルキーマップの除外リスト（exclude）

- ステータス: 承認（オーナー指示による v0.2 機能の前倒し導入）
- 日付: 2026-07-18
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

fakeymacs の `not_emacs_target`（Emacs バインドを適用**しない**アプリの列挙）に相当する機能がないと、「基本は全アプリ、ただし独自キー処理を持つアプリは除く」という実運用の構成が書けない。v0.1 のホワイトリスト方式（適用先を列挙）では、日常使用する全アプリの列挙が必要になり現実的でない。

## 決定

`[[keymap]]` セクションに任意フィールド `exclude`（文字列配列）を追加する。

- **`application = ["*"]` のときのみ指定可**。具体名リストとの併用は設定エラー（除外したいなら list に載せなければよく、意味が重複するため）
- マッチングは exe 名の ASCII 大文字小文字非依存・完全一致（`application` と同一規則）
- 除外付きでもキーマップの優先順位クラスは「グローバル」のまま（アプリ別 > グローバルの規則は不変）

## 理由

- fakeymacs からの移行に必須の表現力を最小の仕様追加（フィールド 1 つ）で得られる
- ブラックリスト方式は「新しくインストールしたアプリにも既定でバインドが効く」という fakeymacs ユーザーの期待と一致する

## 却下した代替案

- **恒等リマップによる素通し**（アプリ別に `"C-h" = "C-h"` を列挙）: キー数 × アプリ数の組合せ爆発になり、また注入イベント経由になるため素通しと完全には等価でない。個別キーの除外（ブラウザの C-t 等）用途には引き続き有効
- **具体名リストへの exclude 併用許可**: 「載せたのに除く」という矛盾した設定を許すことになるためエラーとした
- **正規表現・glob によるマッチ**: 必要になった時点で別 ADR。完全一致で当面の用途は足りる
