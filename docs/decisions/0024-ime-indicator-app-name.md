# ADR 0024: インジケーターへのアプリ名表示（opt-in、exe 名のみ）

- ステータス: 承認（オーナー要望）
- 日付: 2026-07-19
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

オーナーから「インジケーターの下に、IME がオンになったアプリ名を表示できないか。デメリットが大きければ教えてほしい」との要望。デメリットを評価した上で採用を決定した。

## 決定

1. `[ime_indicator]` に `show_app_name`（真偽値、既定 false）を追加する
2. 表示するのは**前面プロセスの exe 名（小文字ベース名。例 `chrome.exe`）**とし、ウィンドウタイトルは表示しない
3. 描画: 「あ」の正方形パネルの下にラベル帯（高さ size の 3/10）を足し、名前の幅に応じてパネル幅を size〜size×2.5 の範囲で自動調整。収まらない分は省略記号（`DT_END_ELLIPSIS`）
4. exe 名の取得は `window.rs` の既存ロジック（`QueryFullProcessImageNameW` → 小文字ベース名）を共有関数化して再利用する。indicator スレッド上での呼び出しでありフック制約は受けない

## 評価したデメリット（採用可能と判断した理由）

- **視覚ノイズの増加**: パネルが縦横に少し大きくなる → 既定 OFF の opt-in とし、使わない人には変化ゼロ
- **長い exe 名の切り詰め**: `windowsterminal.exe` 等は幅上限で省略され得る → 幅の自動調整＋省略記号で実用上判別可能
- **情報量の妥当性**: パネルは対象ウィンドウの中央に出るため、どのアプリかは位置からも分かる（価値は補助的）→ それでも「どのアプリで IME がオンになったか」の明示的な確認手段としてオーナーが有用と判断
- **性能**: 表示時のみ 1 回のプロセス名照会と文字幅計測が増える。indicator スレッド上で µs〜ms オーダーであり、フック・リマップへの影響はない

大きなデメリットは無いと結論した。

## 却下した代替案

- **ウィンドウタイトルの表示**: 開いている文書名・サイト名など機密になり得る情報を画面中央に出してしまう。タイトルは変動も激しく、「キー入力内容や作業内容を表示・記録しない」というプロジェクトの安全方針にも沿わないため exe 名のみとした
- **常時表示（設定なし）**: 表示を大きくしたくない利用者に選択肢がなくなる。既定 OFF の opt-in とした
- **`config.toml` の `application` 一致名の表示**: マッチしたキーマップ名を出す案。IME インジケーターはリマップと独立した機能であり、キーマップとの結合を持ち込まない
