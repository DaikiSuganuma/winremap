# ADR 0006: 注入イベントの識別マーカー（dwExtraInfo 2 種）

- ステータス: 承認
- 日付: 2026-07-18
- 作成: Claude Code（AI モデル: claude-fable-5）

## 文脈

フックは `LLKHF_INJECTED` により注入イベントを素通しする（AGENTS.md 不変条件 1）。しかしフック内部の**論理修飾キー状態の追跡**に関して、注入イベントを一律に無視すると `"CapsLock" = "LCtrl"` のような単キールールで注入した LCtrl がチョード判定に反映されず、「CapsLock を Ctrl として C-h を押す」が機能しない。一方、一時解除・復元（ADR 0005）の補正イベントまで追跡すると、解除した瞬間に「Ctrl は押されていない」ことになり復元判定が壊れる。

## 決定

`SendInput` の `dwExtraInfo` に用途別の 2 マーカーを設定する（`sender.rs` に定数として一元定義）。

| マーカー | 値 | 用途 | 修飾キー追跡 |
|---|---|---|---|
| `MARKER_REMAP` | `0x57524D00` | リマップの出力（置換キー、単キールールで注入した修飾キー） | **追跡する** |
| `MARKER_COMPENSATION` | `0x57524D01` | 一時解除・復元の補正イベント | **追跡しない** |

- フックの追跡対象: 非注入（物理)イベント + `MARKER_REMAP` の注入イベント
- 素通し判定はマーカーに関係なく `LLKHF_INJECTED` のみで行う（他ソフトの注入も含め、注入イベントは一切変換しない）

## 理由

追跡状態の意味を「**ユーザーが論理的に押しているキー**」に統一するため。単キールールで注入した修飾キーはユーザーの意図した押下（追跡すべき）、補正イベントはアプリから見える状態の一時操作（追跡すべきでない）と、性質が異なる。1 つのマーカーではこの区別ができない。

## 却下した代替案

- **マーカーなし（LLKHF_INJECTED のみ）**: 自他の注入を区別できず、上記の追跡が成立しない
- **単一マーカー**: 追跡する／しないのどちらに倒しても、チョード形成か復元判定のどちらかが壊れる（文脈参照）
- **注入前に追跡状態を直接書き換える（イベントを経由しない）**: 注入イベントが実際に配送されたかと状態がずれる可能性があり、フック外から thread_local に触れない制約とも相性が悪い
