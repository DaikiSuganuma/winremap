## 1. デザインツールへの指示文（コピー用）

```
Windows のタスクトレイ（通知領域）用アイコンをデザインしてください。

## 対象アプリ
WinRemap — Rust 製のキーボードリマッパー（キー入力を別のキーに置き換える常駐ツール）。
モチーフ案: キーキャップ、キーボード、置き換えを表す矢印（⇄）など。

## 技術制約（必須）
- 実表示は 16x16px 相当と極小。16px で判別できるシンプルなシルエットにする
- 背景は透過。Windows のタスクバーはライト/ダーク両テーマがあるため、
  どちらの上でも視認できる配色にする（白背景・黒背景の両方でプレビューして確認）
- 細線・グラデーション・文字は 16px では潰れるので使わない。
  塗りは 1〜2 色のフラットデザイン
- スタイルは Windows 11 の Fluent 系アイコンに馴染むもの

## 状態バリエーション（必須）
1. 有効状態: カラー版
2. 無効状態: 同じ形のグレースケール版（一目で無効と分かること）

## 納品形式
- SVG（マスター）
- PNG: 16 / 24 / 32 / 48 / 256 px（透過、各状態）
- 可能なら .ico（16/24/32/48/256 を内包、exe 本体のアイコン用）
```

## 2. 納品物の受け入れと組み込み

### 最小要件

**32x32 の透過 PNG 2 枚（有効/無効）** があれば組み込み可能。フル納品（SVG + 各サイズ + .ico）は exe 本体アイコンまで揃えたい場合のみ。

### 選定時のチェックポイント

- **16px に縮小した状態で比較する**。原寸で良く見えても 16px で潰れる案は不採用
- ライト/ダーク両方のタスクバー上でのプレビュー確認
- 無効（グレー）状態が一目で区別できること

### 組み込み手順（エージェント作業）

1. 画像を `assets/`（新設）に配置
2. `src/tray.rs` の `build_icon`（現在はコード生成の 32x32 RGBA）を、`include_bytes!` で埋め込んだアセット読み込みに差し替える
3. PNG デコードの方法（png クレート追加 or ビルド時に生 RGBA へ変換して埋め込み）はその時点で ADR（Architecture Decision Record: 設計判断の記録）を書いて判断する
4. .ico がある場合は exe リソースへの埋め込み（winresource 等）も同 ADR で判断する

### 組み込み状況

**2026-07-18 組み込み完了**。オーナーデザインのキーボードアイコン（青/グレー）を `assets/` に受領し、`build.rs` + winresource で exe リソースとして埋め込み（トレイと exe 本体アイコンを兼用）。方式の詳細は [ADR 0010](./decisions/0010-icon-embedding.md)。それまでのコード生成アイコン（青い角丸＋白キーキャップ 6 個）は履歴のみに残る。
