## 0. 本書の範囲と原則

- B1（閲覧）で確定した体裁（表の罫線・反転見出し・記法ペイン・ヘッダー帯）を**そのまま使い**、編集のための要素だけを足す。新しい見た目の言語を作らない
- 画面は**閲覧モード**と**編集モード**の 2 状態。同時に両方は無い（ADR 0049）
- モーダル OS ダイアログは使わない。確認・エラーは**ウィンドウ内のバー／インライン**で出す（v0.2 §5）。唯一 `notify::error` に回すのは「GUI 自体が出せない致命」だけ
- 寸法・色は既存の `src/theme.rs` 定数に紐づける（新規は §8 に列挙）

### 0.1 v0.2 §3 からの寸法の更新

| 項目 | v0.2 §3 の記載 | 現行実装（本書が採用） |
|---|---|---|
| ウィンドウ初期サイズ | 960x860 | **1120x860**（`theme::CONFIG_WINDOW`。記法ペインを収めるため広げた） |

| ヘッダー内ファイル表の幅 | — | 利用可能幅 × `FILE_TABLE_WIDTH_RATIO`(0.5) |

ウィンドウサイズ・位置は記憶しない（毎回この大きさ。v0.2 §9-3 据え置き）。

## 2. 閲覧モード（B1＋［編集］）

B1 の画面に、ヘッダーへ **［編集］ボタン 1 個**を足すだけ。詳細ペインは現行のまま（読み取り専用のラベル・表）。

```
┌─ [config-header]（帯色 = chrome_frame 通常） ─────────────────────────────┐
│ 🗎 設定ファイル                                        WinRemap v0.4.0      │
│                                                                           │
│ ┌─ 項目 ───────┬─ 値 ─────────────────────┐   ↕ 設定を再読み込み          │
│ │ パス          │ …\winremap\config.toml   │   ↗ テキストエディタで開く    │
│ │ 更新日時      │ 2026-07-22 09:12:04      │   ✎ 編集                     │
│ │ 読み込み日時  │ 2026-07-22 09:00:41      │                              │
│ └───────────────┴──────────────────────────┘                             │
│ この画面は表示専用です。値を変えるには［編集］を押してください。            │
└───────────────────────────────────────────────────────────────────────────┘
   （左一覧・詳細・記法ペインは現行 B1 のまま）
```

- ［編集］は右列の操作群（現行「テキストエディタで開く」「再読み込み」の並び）へ**3 つ目として追加**。アイコンは `pencil`（Bootstrap Icons、白焼き→本文色着色。ADR 0042）
- 現行の但し書き `config_window_readonly`（「この設定ウィンドウは表示専用です」）は、文言を「値を変えるには［編集］を押してください」に寄せて残す（閲覧モードでのみ表示）
- ［編集］押下 → ファイルを読んでドラフト生成 → 編集モードへ（読めなければモード遷移せず、フッターに理由。§6.3）

## 3. 編集モード

### 3.1 ヘッダー（編集中）

帯の色を「編集中」色に変え（`theme::edit_mode_frame` 新設・§8）、操作を［保存］［元に戻す］へ差し替える。ファイル情報の表は残す（今どのファイルを編集しているかは編集中こそ要る）。

```
┌─ [config-header]（帯色 = 編集中色） ──────────────────────────────────────┐
│ 🗎 設定ファイル                                        WinRemap v0.4.0      │
│                                                                           │
│ ┌─ 項目 ───────┬─ 値 ─────────────────────┐   💾 保存                     │
│ │ パス          │ …\winremap\config.toml   │   ↶ 元に戻す                 │
│ │ 更新日時      │ 2026-07-22 09:12:04      │                              │
│ │ 読み込み日時  │ 2026-07-22 09:00:41      │                              │
│ └───────────────┴──────────────────────────┘                             │
│ ● 編集中 — ［保存］を押すまで config.toml は変わりません。                 │
└───────────────────────────────────────────────────────────────────────────┘
```

- ［保存］= §4 の保存フロー。検証を通れば書き込み→リロード→閲覧モードへ。通らなければフッターに件数（§5.4）
- ［元に戻す］= ドラフト破棄→閲覧モードへ（未保存分は捨てる。押下時に確認は出さない＝明示操作なので）
- 「● 編集中 —」の行は帯内に常時。閲覧の但し書きの位置を置き換える

### 3.2 詳細ペイン — キーマップ（編集中）

B1 の各セクション（名前／対象アプリ／除外／リマップ規則）を編集可能ウィジェットに替える。**セクションの並び・見出し・記法ペインは B1 と同一**。

```
┌ [CentralPanel] キーマップ「emacs」 ──────────────────────┐ ┌ 記法 ───────┐
│                                                          │ │ ⌨ キー記法   │
│  名前  name = [ emacs                              ]      │ │ C-  Ctrl    │
│  # Emacs 風        ← 既存コメントは薄字で表示（読取専用）  │ │ A-  Alt     │
│                                                          │ │ S-  Shift   │
│  ▤ 対象アプリ  application                                │ │ W-  Win     │
│  ┌ アプリ ──────────────┬ コメント ──────────┬───┐        │ │             │
│  │ [ Chrome.exe        ]│ ブラウザ           │ ✕ │        │ │ 空白でシーケ │
│  │ [ code.exe          ]│                    │ ✕ │        │ │ ンス。カンマ │
│  └──────────────────────┴────────────────────┴───┘        │ │ でマクロ。   │
│  [ ＋ アプリを追加 ]  [ ⊕ 今の前面アプリから取得 ]         │ │ 🔗 ヘルプ    │
│  ⓘ exe 名は大文字小文字を区別しません                     │ └─────────────┘
│                                                          │
│  ▤ リマップ規則  [keymap.remap]                           │
│  ┌ WinRemap 入力 ─┬ アプリ向け出力 ─┬ 他のキーマップ ┬ コメント ┬───┐
│  │ [ C-h        ] │ [ Back         ]│ global         │ 削除     │ ✕ │
│  │   ⓘ Ctrl + h   │   ⓘ Back        │                │          │   │
│  │ [ A-x h      ] │ [ C-z          ]│                │ 2 スト   │ ✕ │
│  │   ⓘ Alt+x → h  │   ⓘ Ctrl + z    │                │          │   │
│  │ [ C-t        ] │ [ C-Right, C-Left]│              │ マクロ   │ ✕ │
│  │   ⓘ Ctrl + t   │   ⓘ C-Right → C-Left（マクロ）    │          │   │
│  └────────────────┴─────────────────┴────────────────┴──────────┴───┘
│  [ ＋ 規則を追加 ]                                        │
└──────────────────────────────────────────────────────────┘
```

要素の対応:

| 画面要素 | ウィジェット | 備考 |
|---|---|---|
| 名前 | `TextEdit`（単一行） | 空文字可（無名キーマップ）。左一覧のラベルは B1 の `keymap_label` に従い、無名なら対象アプリ表示 |
| 既存コメント（`# …`） | 薄字ラベル（読取専用） | コメントは GUI で編集しない（保存時に `toml_edit` が保持）。B1 の `note()` を流用 |
| 対象アプリ 各行 | `TextEdit` ＋ 行末 ✕ | 表は B1 の `table()` を編集セル版に拡張。コメント列は薄字ラベル（読取専用） |
| ＋ アプリを追加 | ボタン | 末尾に空行を足す |
| ⊕ 今の前面アプリから取得 | ボタン | §5.3。`application` が `["*"]` の行には出さない（全アプリ指定に個別 exe は無意味） |
| リマップ規則 各行 | 入力 `TextEdit`／出力 `TextEdit`／✕ | 各欄の直下に逐次検証の解釈（ⓘ）／エラー（§5.1・§5.2） |
| 他のキーマップ列 | 薄字ラベル（読取専用） | B1 の `shared_inputs` をドラフトで再計算。該当 0 なら列ごと非表示（現行踏襲） |
| コメント列 | 薄字ラベル（読取専用） | 同上 |
| ＋ 規則を追加 | ボタン | 末尾に空の入力/出力行を足す |

- **除外（exclude）** は `application == ["*"]` のキーマップでのみ、対象アプリの下に同じ体裁の編集表として出す（B1 の条件を踏襲）
- 入力・出力セルは**書かれた綴りをそのまま編集**（正準化しない。ADR 0049 決定 5）。B1 は正準化後（`C-S-h`）を表示していたが、編集モードではドラフトが持つ生綴りを出す
- ⓘ 行（解釈のプレビュー）は妥当なときだけ出す。エラー時は §5.2 の赤字に替わる

### 3.3 左一覧（編集中）

B1 の一覧に、キーマップの増減・並べ替えボタンを足す。

```
┌ [config-list] ──────────┐
│  ⚙ 全体設定             │   ← 選ぶと §4.3
│                         │
│  キーマップ              │
│   ▸ notepad             │
│   ▸ emacs      ◀ 選択中 │
│   ▸ *（全アプリ）        │
│                         │
│  [＋追加] [－削除]        │   ← 編集モードのみ
│  [ ↑ ]   [ ↓ ]          │   ← 選択中キーマップを上下に移動
└─────────────────────────┘
```

- ＋追加＝末尾に空のキーマップ（`application = []`、規則なし）を足して選択。－削除＝選択中を消す（確認はフッターに「削除しました／元に戻すには［元に戻す］」を出す方式か、直接削除か → §9 未決）
- ↑↓＝選択中キーマップの `[[keymap]]` 順を入れ替え。ファイル上でもテーブルごと移動（§4.2 の並べ替え）。ドラッグは使わない（v0.2 §3.1）
- 「全体設定」には増減ボタンは効かない（キーマップ選択時のみ活性）

### 3.4 ドラフトと画面の関係（再掲・02 §3）

編集モード中に表示するのは**ドラフト**。裏で `REMAP_TABLE` が差し替わっても画面は動じない（フックは無関係）。記憶マクロ表示（§4.3）だけはライブの実行時状態を読む。

## 4. セクション別レイアウト詳細

### 4.1 表の編集セル（B1 `table()` の拡張）

B1 の `table()`（罫線・反転見出し・縞・セル余白）を維持し、セルの中身をラベルから `TextEdit` に替えられるようにする。列幅・折り返しの規則（コメント列は `wrap`）は現行のまま。編集セルは等幅フォント（入力がキー記法・exe 名のため）。

### 4.2 削除・追加時のコメントの扱い（画面には出ないが挙動として明示）

- 行の ✕（削除）→ ドラフトから該当行を除く。**保存時**に `toml_edit` がその項目と密着コメントを消す（§02 §4.2）。画面上、削除は即時（ドラフト反映）だがファイルは保存まで不変
- ＋追加 → 末尾に空行。保存時にテーブル／配列末尾へ挿入

### 4.3 詳細ペイン — 全体設定（編集中）

記法ペインは出さない。`[macro]` と `[ime_indicator]` を編集ウィジェットで。B1 の `settings_table` の値セルを入力に替える。

```
┌ [CentralPanel] 全体設定 ─────────────────────────────────────────┐
│                                                                  │
│  ▤ マクロ  [macro]                                                │
│  ┌ 項目 ─────────┬ キー ────────┬ 値 ─────────────┬ コメント ┐    │
│  │ 再生の間隔     │ delay_ms     │ [====|----] 8 ms │ …       │    │
│  │ 記憶開始       │ record_start │ [ S-F10        ] │          │    │
│  │   ⓘ Shift+F10  │              │                  │          │    │
│  │ 記憶終了       │ record_stop  │ [ S-F10        ] │ 省略=開始 │    │
│  │ 再生           │ record_play  │ [ F10          ] │          │    │
│  └────────────────┴──────────────┴──────────────────┴──────────┘    │
│  ⓘ 記憶キーを空にすると機能は無効になります（3 つとも空で完全 OFF）  │
│                                                                  │
│  （記憶中のマクロ ＝ ライブ状態。編集の対象外。B1 の枠をそのまま） │
│  ┌──────────────────────────────────────────────────┐            │
│  │ 記憶中のマクロ   C-a → C-k → C-y                    │            │
│  └──────────────────────────────────────────────────┘            │
│  ※ これはメモリ上の記憶で、config.toml には保存されません          │
│                                                                  │
│  ▤ IME インジケーター  [ime_indicator]                            │
│  有効        enabled       [✓]                                    │
│  表示時間    duration_ms   [======|--] 1200 ms                    │
│  大きさ      size          [===|-----] 64                         │
│  不透明度    opacity       [=======|-] 220                        │
│  アプリ名表示 show_app_name [✓]                                   │
│  切替キー    trigger_keys  [ C-Space                        ]     │
│                ⓘ Ctrl + Space                                    │
└──────────────────────────────────────────────────────────────────┘
```

- `delay_ms` / `duration_ms` / `size` / `opacity` は `Slider`（範囲は `keymap`／`ime_indicator_settings` の MIN/MAX 定数から。0-15 / 範囲は既存検証と一致）。数値も併記
- `enabled` / `show_app_name` は `Checkbox`（アイコンは付けない。B1・ADR 0040 の判断踏襲）
- `record_start/stop/play` と `trigger_keys` はキー記法 `TextEdit`＋逐次検証（§5）。`trigger_keys` は配列なのでカンマ区切り（複数可）
- IME の詳細スライダー群は `enabled` が ✓ のときだけ出す（B1 の条件踏襲）。マクロ記憶行は 3 キーのどれかが埋まっているときに意味を持つが、編集中は常に 3 行出す（設定しに来ているため）
- v0.1 綴り `macro_delay_ms` のファイルは、キー列にその綴りを出し、保存時もその綴りで書く（ドラフトの `uses_legacy_delay_key`。02 §3.2）

## 5. キー記法の入力・検証・入力補助

### 5.1 妥当なとき（解釈のプレビュー）

入力欄の**直下**に、解釈を人間語で薄字表示。

```
 入力  [ A-x h                    ]
         ⓘ Alt+x に続けて h（2 ストローク）
```

例: `C-h`→「Ctrl + h」、`A-x h`→「Alt+x に続けて h（2 ストローク）」、出力の `C-Right, C-Left`→「C-Right → C-Left（マクロ）」。

### 5.2 不当なとき（理由と候補）

欄の枠を警告色にし、直下に理由＋候補（あれば 1 つ）。

```
 入力  [ Bak                      ]   ← 枠は警告色
         ⚠ 未知のキー名 "Bak"。"Back" のことですか？
```

- 候補は既知キー名との近さ（編集距離）で最も近い 1 つ。無ければ理由のみ
- この状態でも入力は続けられる（打鍵の途中で一時的に不当になるのは正常）。保存は §5.4 で止まる

### 5.3 「今の前面アプリから取得」（3 秒カウントダウン）

ボタン押下後、その場でカウントダウン表示に変わり、対象アプリへ切り替える猶予を与える。

```
 押す前:  [ ⊕ 今の前面アプリから取得 ]

 押した後: [ ⏳ 3… 対象アプリを前面にしてください ]  → 2… → 1…
           → 取得: window::with_foreground_exe で前面 exe 名を application に追加
           → ボタンは元の表示へ戻る
```

- カウントダウン中はボタンを再押下不可。取得後、重複する exe 名なら足さない
- 取得元は既存の安全な公開関数（GUI に unsafe を持ち込まない。02 §6.1）

### 5.4 検証エラーのフッターバー（保存を押したとき）

保存前検証（`config::parse_str`）が落ちたら、`config-footer` に件数と先頭 1 件。クリックで該当欄へスクロール（v0.2 §5、02 §4.3）。

```
┌ [config-footer]（警告色） ────────────────────────────────────────────────┐
│ ⚠ 2 件の問題があります: 12 行目 未知のキー名 "Bak"（"Back"?）  ［次へ ▸］    │
└───────────────────────────────────────────────────────────────────────────┘
```

- ［次へ▸］で 2 件目へ。各件クリックで対応する詳細欄までスクロール＆ハイライト（逆写像の粒度は 02 §9 未決 1。当面は「先頭の該当欄まで」で可）
- 検証を通れば保存実行→フッターは消える

### 5.5 キー名一覧（`?`）

入力欄脇の `?` を押すと、使用可能キー名の一覧を小ポップアップ（`egui::popup`／子ウィンドウ）で表示。`keymap` のキー名テーブルから生成（表示専用）。

```
 入力  [ C-              ] [?]
                          └─▶ ┌ 使えるキー名 ─────────────┐
                              │ 修飾: C- A- S- W-          │
                              │ 文字: a-z 0-9              │
                              │ 特殊: Back Enter Esc Tab   │
                              │       Space Delete Home …  │
                              │ ファンクション: F1-F24     │
                              │ 方向: Left Right Up Down    │
                              └────────────────────────────┘
```

## 6. 外部変更・保存失敗・未保存クローズ（すべてウィンドウ内で）

OS モーダルは使わない。`config-footer`（またはヘッダー下の帯）へ出す。

### 6.1 外部変更の検出（保存直前）

開いた時点の `(mtime, size)` と保存直前が食い違うとき、保存を止めて選択を促す。

```
┌ [config-footer]（注意色） ────────────────────────────────────────────────┐
│ ⚠ config.toml が WinRemap の外で変更されています。                          │
│    ［上書き保存］  ［読み直す（編集を破棄）］  ［キャンセル］                │
└───────────────────────────────────────────────────────────────────────────┘
```

- ［上書き保存］= ドラフトを保存（外部変更は失われる）。［読み直す］= ドラフト破棄して再読込＝編集モードを作り直す。［キャンセル］= 何もしない（編集継続）
- 黙って上書きしない（02 §6.2）

### 6.2 保存失敗（権限・ディスク）

```
┌ [config-footer]（警告色） ────────────────────────────────────────────────┐
│ ⚠ 保存できませんでした: アクセスが拒否されました（config.toml）。           │
└───────────────────────────────────────────────────────────────────────────┘
```

編集モードは維持（内容は失われない）。ダイアログにしない（GUI が前面にある。v0.2 §5）。

### 6.3 編集に入れない／未保存で閉じる

- ［編集］押下でファイルが読めない: モード遷移せず、フッターに「編集を開始できません: <理由>」
- 未保存のまま窓を閉じる（×＝隠す。ADR 0032）: ヘッダー下に確認帯を出す（§9 未決）。既定案は「保存せずに閉じますか? ［閉じる］［キャンセル］」。閉じるとドラフト破棄

## 7. 状態遷移

```
        ［編集］(ファイル読込 成功)
 閲覧 ───────────────────────────────▶ 編集
  ▲   ◀───────────────────────────────  │
  │        ［元に戻す］/ 保存成功後       │
  │                                     │ ［保存］
  │                              ┌──────┴───────┐
  │                     検証NG   │  保存前検証   │  検証OK
  │                    ┌─────────┴──┐        ┌──┴─────────┐
  │                    ▼            │        ▼            │
  │            フッターに件数        │   外部変更?         │
  │            （編集継続）          │   ├ 無 → 原子的保存 → request_reload → 閲覧へ
  │                                 │   └ 有 → §6.1 の選択（上書き/読直し/キャンセル）
  └─ トレイ「再読み込み」: 閲覧なら表示更新 / 編集ならライブ表のみ更新・ドラフト不変
```
