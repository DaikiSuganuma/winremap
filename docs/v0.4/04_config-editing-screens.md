## 0. 本書の範囲と原則

- **レイアウトの参照モデルは Windows エクスプローラー**（オーナー指示 2026-07-22）。上＝アドレスバー、左＝ツリー型ナビゲーション、中央＝パンくず＋詳細、下＝ステータスバー
- B1 で確定した**表の体裁**（罫線・反転見出し・記法ペイン）はそのまま使い、周囲の骨格をエクスプローラー風に組み替える
- 画面は**閲覧モード**と**編集モード**の 2 状態。同時に両方は無い（ADR 0049）
- モーダル OS ダイアログは使わない。確認・エラーは**ウィンドウ内のバー／インライン**で出す（v0.2 §5）。唯一 `notify::error` に回すのは「GUI 自体が出せない致命」だけ
- 寸法・色は既存の `src/theme.rs` 定数に紐づける（新規は §9 に列挙）

## 1. ウィンドウの骨格（両モード共通）

```
┌─ WinRemap 設定 ───────────────────────────────────────────── 1120 × 860 ─┐
│ [config-header]   Panel::top … アドレスバー＋操作＋バージョン（§2）        │
├───────────┬──────────────────────────────────────────┬───────────────────┤
│[config-   │ [CentralPanel]                           │ [config-notation] │
│ list]     │  パンくず（§4.1）                         │  記法凡例         │
│ ナビゲー  │  ──────────────────────                  │  （キーマップ選択  │
│ ション    │  詳細ペイン                               │   時のみ・240px）  │
│ ツリー    │  ・キーマップ → §4.2 / §4.3               │  ScrollArea       │
│ 220px     │  ・全体設定   → §4.4（記法ペイン無）       │                   │
│ §3        │  ScrollArea vertical                     │                   │
├───────────┴──────────────────────────────────────────┴───────────────────┤
│ [config-footer]    Panel::bottom … 検証・確認バー（編集モードのみ・§6/§7）  │
├───────────────────────────────────────────────────────────────────────────┤
│ [config-statusbar] Panel::bottom … ステータスバー（常時・§5）              │
└───────────────────────────────────────────────────────────────────────────┘
```

- `config-statusbar` は**常時**出す新規パネル（両モード・最下部）
- `config-footer` は**編集モードのみ**、ステータスバーの直上に出す検証・確認バー。閲覧モードでは出さない
- 記法ペインは「キーマップを表示していて、かつ規則がある」ときだけ（`shows_rules`、現行踏襲）。全体設定選択時は出さない
- B1 のヘッダーにあった**ファイル情報表（パス・更新日時・読み込み日時）と「表示専用です」の注記は廃止**（オーナー指示）。ファイルの所在はアドレスバー（§2）が、鮮度は変更ありマーク（§2.2）とステータスバーのメッセージ（§5）が引き受ける

## 2. ヘッダー ＝ アドレスバー

### 2.1 閲覧モード

```
┌─ [config-header]（帯色 = chrome_frame 通常） ─────────────────────────────┐
│ 📁 %APPDATA%\winremap  ›  [ config.toml            ▾ ]  ✎  ↻      v0.4.0 │
└───────────────────────────────────────────────────────────────────────────┘
   └ フォルダーパス（表示のみ）  └ ファイル名プルダウン      │  │       └ バージョン情報（右端・weak）
                                                        │  └ ↻ 設定を再読み込み
                                                        └ ✎ 編集（編集モードへ）
```

- **フォルダーパス**は現行設定ファイルの親フォルダー（既定 `%APPDATA%\winremap`、`-c` 起動時はそのフォルダー）。表示のみで変更不可（フォルダーの変更は起動オプションのままとする → §11）
- **ファイル名プルダウン**（`egui::ComboBox`）は、そのフォルダー直下の `*.toml` を列挙し、選択で設定ファイルを切り替える（§2.3）
- ✎（編集）・↻（再読み込み）はアドレスバーの右隣にアイコンボタンで置く（オーナー指示）。ツールチップに名称
- 右端に**バージョン情報**（`WinRemap v{CARGO_PKG_VERSION}`、weak）。B1 の右端表示を踏襲
- B1 の「テキストエディタで開く」は、プルダウンの最下段へアクション項目として移す（§2.3。ヘッダーのアイコンを 2 つに保つため → §11 未決 5）

### 2.2 変更ありマーク（●）

ファイルの中身がこの画面の読み込み済み状態と食い違ったら、ファイル名の右に **●** を付ける。

```
│ 📁 %APPDATA%\winremap  ›  [ config.toml ●          ▾ ]  ✎  ↻      v0.4.0 │
```

- 判定は現行 `ConfigWindow::file_time` の機構（`FILE_TIME_INTERVAL` = 2 秒間隔の `(mtime, size)` ポーリング）を転用。**毎フレームのディスクアクセスはしない**（現行コメントの通り）
- 使用中ファイル: WinRemap が最後に読み込んだ時点の `(mtime, size)` と現在が異なれば ●。↻（再読み込み）または保存で消える
- プルダウン内のその他ファイル: 一覧を前回組み立てた時点との差分で ●（§2.3）
- 廃止した「更新日時 vs 読み込み日時」の表が伝えていた**「保存したがまだ読み込まれていない」状態**は、この ● が引き継ぐ

### 2.3 ファイル名プルダウンの中身

```
   [ config.toml ●          ▾ ]
   ┌───────────────────────────────┐
   │ ✓ config.toml            ●    │  ← ✓ = 使用中、● = 変更あり
   │   config-backup.toml          │
   │   experiment.toml        ●    │
   ├───────────────────────────────┤
   │ ↗ テキストエディタで開く        │  ← 使用中ファイルを開く（B1 から移設）
   │ 📁 フォルダーを開く             │  ← エクスプローラーでフォルダーを開く
   └───────────────────────────────┘
```

- 一覧はプルダウンを**開いた時点**でフォルダーを読み直して組み立てる（開閉ごとに 1 回。ポーリングしない）。ファイル名昇順
- 使用中ファイルには ✓（行頭）。各行の ● は前回一覧時点からの `(mtime, size)` 差分
- **別ファイルを選択**すると: パスを切り替え → 再読み込み（`request_reload` と同じ経路）→ ステータスバーに「読み込み完了しました」（§5）。切替に失敗（読めない・検証エラー）したときの挙動は §10-1
- `*.toml` が使用中ファイル 1 つだけでもプルダウンは出す（アクション項目 2 つがあるため空にならない）

### 2.4 編集モード

帯色を「編集中」色（`theme::edit_mode_frame` 新設・§9）に変え、操作アイコンを差し替える。

```
┌─ [config-header]（帯色 = 編集中色） ──────────────────────────────────────┐
│ 📁 %APPDATA%\winremap  ›  [ config.toml            ▾ ]  💾  ↶     v0.4.0 │
└───────────────────────────────────────────────────────────────────────────┘
                            └ プルダウンは無効化           │  └ ↶ 元に戻す
                                                          └ 💾 保存
```

- ✎ ↻ → **💾（保存）↶（元に戻す）** に差し替え
- **プルダウンは無効化**（編集中のファイル切替は編集内容の行き先を失うため。ツールチップ「編集中はファイルを切り替えられません」）
- ［保存］= 02 §4 の保存フロー。検証を通れば書き込み→リロード→閲覧モードへ。通らなければフッターに件数（§6.4）
- ［元に戻す］= ドラフト破棄→閲覧モードへ（未保存分は捨てる。押下時に確認は出さない＝明示操作なので）
- 「編集中」の告知はパンくず行の右端（§4.1）に置く（B1 ヘッダー注記の置き換え）

## 3. 左ナビゲーション（サイドバー）

アイコン付きのツリー風表示にする（**開閉機能はなし**・オーナー指示）。

```
┌ [config-list] 220px ────────┐
│  ⚙ 全体設定                 │
│  ⌨ キーマップ                │   ← 見出し（選択不可・strong）
│ ░░░ 🗔 notepad ░░░░░░░░░░░ │   ← 選択中: 行全体を灰色で塗る
│      🗔 emacs               │
│      🗔 *（全アプリ）        │
│                             │
│  [＋追加] [－削除]           │   ← 編集モードのみ（§3.1）
│  [ ↑ ]   [ ↓ ]             │
└─────────────────────────────┘
```

- **アイコン**: 全体設定 = `gear`、キーマップ見出し = `keyboard`、各キーマップ = 既存 `Icon::Apps`（window-stack。キーマップ＝アプリ集合の対応）
- キーマップの行は見出しから 1 段インデント（ツリーの階層表現。開閉マークは付けない）
- **選択行は行全体（サイドバー全幅）を灰色で塗る**（エクスプローラーの選択と同じ・オーナー指示）。既存の `selectable_value`（テキスト幅のみのハイライト）は使わず、全幅の選択行ウィジェットにする。塗り色は `theme::sidebar_selection_fill`（新設・§9。ライト＝明るい灰、ダーク＝暗い灰）
- ラベルは B1 の `keymap_label` 踏襲（無名キーマップは対象アプリ表示）

### 3.1 編集モードの増減・並べ替え

- ＋追加＝末尾に空のキーマップ（`application = []`、規則なし）を足して選択。－削除＝選択中を消す（確認の有無 → §11 未決 1）
- ↑↓＝選択中キーマップの `[[keymap]]` 順を入れ替え。ファイル上でもテーブルごと移動（§4.5 の並べ替え）。ドラッグは使わない（v0.2 §3.1）
- 「全体設定」「キーマップ見出し」には増減ボタンは効かない（キーマップ選択時のみ活性）

## 4. 中央ペイン

### 4.1 パンくずリスト（中央ペインの先頭・両モード）

中央ペインの最上部に、選択中の場所をパンくずで示す（オーナー指示）。

```
 閲覧:  設定 > 全体設定
        設定 > キーマップ > emacs

 編集:  設定 > キーマップ > emacs        ● 編集中 — ［保存］を押すまで config.toml は変わりません
```

- 区切りは「>」。表示のみ（クリックでの移動は付けない。左ナビゲーションがあるため → §11 未決 6）
- 無名キーマップは `keymap_label` の表示（対象アプリ名）をそのまま使う
- **編集モードでは右端に「● 編集中 —」の告知**（B1 ヘッダーの但し書きの後継。帯色と合わせて二重に効かせる）
- パンくずの下に罫線（`ui.separator()`）を引き、詳細ペインと区切る

### 4.2 詳細ペイン — キーマップ（閲覧）

パンくずの下は現行 B1 のまま（読み取り専用のラベル・表・記法ペイン）。変更なし。

### 4.3 詳細ペイン — キーマップ（編集中）

B1 の各セクション（名前／対象アプリ／除外／リマップ規則）を編集可能ウィジェットに替える。**セクションの並び・見出し・記法ペインは B1 と同一**。

```
┌ [CentralPanel] ──────────────────────────────────────────┐ ┌ 記法 ───────┐
│ 設定 > キーマップ > emacs      ● 編集中 — ［保存］を…      │ │ ⌨ キー記法   │
│ ──────────────────────────────────────────────────────── │ │ C-  Ctrl    │
│  名前  name = [ emacs                              ]      │ │ A-  Alt     │
│  # Emacs 風        ← 既存コメントは薄字で表示（読取専用）  │ │ S-  Shift   │
│                                                          │ │ W-  Win     │
│  ▤ 対象アプリ  application                                │ │             │
│  ┌ アプリ ──────────────┬ コメント ──────────┬───┐        │ │ 空白でシーケ │
│  │ [ Chrome.exe        ]│ ブラウザ           │ ✕ │        │ │ ンス。カンマ │
│  │ [ code.exe          ]│                    │ ✕ │        │ │ でマクロ。   │
│  └──────────────────────┴────────────────────┴───┘        │ │ 🔗 ヘルプ    │
│  [ ＋ アプリを追加 ]  [ ⊕ 今の前面アプリから取得 ]         │ └─────────────┘
│  ⓘ exe 名は大文字小文字を区別しません                     │
│                                                          │
│  ▤ リマップ規則  [keymap.remap]                           │
│  ┌ WinRemap 入力 ─┬ アプリ向け出力 ─┬ 他のキーマップ ┬ コメント ┬───┐
│  │ [ C-h        ] │ [ Back         ]│ global         │ 削除     │ ✕ │
│  │   ⓘ Ctrl + h   │   ⓘ Back        │                │          │   │
│  │ [ A-x h      ] │ [ C-z          ]│                │ 2 スト   │ ✕ │
│  │   ⓘ Alt+x → h  │   ⓘ Ctrl + z    │                │          │   │
│  │ [ C-t        ] │ [ C-Right, C-Left]│              │ マクロ   │ ✕ │
│  │   ⓘ Ctrl + t   │   ⓘ C-Right → C-Left（マクロ）    │          │   │
│  └────────────────┴─────────────────┴────────────────┴──────────┴───┘
│  [ ＋ 規則を追加 ]                                        │
└──────────────────────────────────────────────────────────┘
```

要素の対応:

| 画面要素 | ウィジェット | 備考 |
|---|---|---|
| 名前 | `TextEdit`（単一行） | 空文字可（無名キーマップ）。左ナビのラベルは `keymap_label` に従い、無名なら対象アプリ表示 |
| 既存コメント（`# …`） | 薄字ラベル（読取専用） | コメントは GUI で編集しない（保存時に `toml_edit` が保持）。B1 の `note()` を流用 |
| 対象アプリ 各行 | `TextEdit` ＋ 行末 ✕ | 表は B1 の `table()` を編集セル版に拡張。コメント列は薄字ラベル（読取専用） |
| ＋ アプリを追加 | ボタン | 末尾に空行を足す |
| ⊕ 今の前面アプリから取得 | ボタン | §6.3。`application` が `["*"]` の行には出さない（全アプリ指定に個別 exe は無意味） |
| リマップ規則 各行 | 入力 `TextEdit`／出力 `TextEdit`／✕ | 各欄の直下に逐次検証の解釈（ⓘ）／エラー（§6.1・§6.2） |
| 他のキーマップ列 | 薄字ラベル（読取専用） | B1 の `shared_inputs` をドラフトで再計算。該当 0 なら列ごと非表示（現行踏襲） |
| コメント列 | 薄字ラベル（読取専用） | 同上 |
| ＋ 規則を追加 | ボタン | 末尾に空の入力/出力行を足す |

- **除外（exclude）** は `application == ["*"]` のキーマップでのみ、対象アプリの下に同じ体裁の編集表として出す（B1 の条件を踏襲）
- 入力・出力セルは**書かれた綴りをそのまま編集**（正準化しない。ADR 0049 決定 5）。B1 は正準化後（`C-S-h`）を表示していたが、編集モードではドラフトが持つ生綴りを出す
- ⓘ 行（解釈のプレビュー）は妥当なときだけ出す。エラー時は §6.2 の赤字に替わる

### 4.4 詳細ペイン — 全体設定（編集中）

記法ペインは出さない。`[macro]` と `[ime_indicator]` を編集ウィジェットで。B1 の `settings_table` の値セルを入力に替える。

```
┌ [CentralPanel] ──────────────────────────────────────────────────┐
│ 設定 > 全体設定                ● 編集中 — ［保存］を…              │
│ ──────────────────────────────────────────────────────────────── │
│  ▤ マクロ  [macro]                                                │
│  ┌ 項目 ─────────┬ キー ────────┬ 値 ─────────────┬ コメント ┐    │
│  │ 再生の間隔     │ delay_ms     │ [====|----] 8 ms │ …       │    │
│  │ 記憶開始       │ record_start │ [ S-F10        ] │          │    │
│  │   ⓘ Shift+F10  │              │                  │          │    │
│  │ 記憶終了       │ record_stop  │ [ S-F10        ] │ 省略=開始 │    │
│  │ 再生           │ record_play  │ [ F10          ] │          │    │
│  └────────────────┴──────────────┴──────────────────┴──────────┘    │
│  ⓘ 記憶キーを空にすると機能は無効になります（3 つとも空で完全 OFF）  │
│                                                                  │
│  （記憶中のマクロ ＝ ライブ状態。編集の対象外。B1 の枠をそのまま） │
│  ┌──────────────────────────────────────────────────┐            │
│  │ 記憶中のマクロ   C-a → C-k → C-y                    │            │
│  └──────────────────────────────────────────────────┘            │
│  ※ これはメモリ上の記憶で、config.toml には保存されません          │
│                                                                  │
│  ▤ IME インジケーター  [ime_indicator]                            │
│  有効        enabled       [✓]                                    │
│  表示時間    duration_ms   [======|--] 1200 ms                    │
│  大きさ      size          [===|-----] 64                         │
│  不透明度    opacity       [=======|-] 220                        │
│  アプリ名表示 show_app_name [✓]                                   │
│  切替キー    trigger_keys  [ C-Space                        ]     │
│                ⓘ Ctrl + Space                                    │
└──────────────────────────────────────────────────────────────────┘
```

- `delay_ms` / `duration_ms` / `size` / `opacity` は `Slider`（範囲は `keymap`／`ime_indicator_settings` の MIN/MAX 定数から。0-15 / 範囲は既存検証と一致）。数値も併記
- `enabled` / `show_app_name` は `Checkbox`（アイコンは付けない。B1・ADR 0040 の判断踏襲）
- `record_start/stop/play` と `trigger_keys` はキー記法 `TextEdit`＋逐次検証（§6）。`trigger_keys` は配列なのでカンマ区切り（複数可）
- IME の詳細スライダー群は `enabled` が ✓ のときだけ出す（B1 の条件踏襲）。マクロ記憶行は 3 キーのどれかが埋まっているときに意味を持つが、編集中は常に 3 行出す（設定しに来ているため）
- v0.1 綴り `macro_delay_ms` のファイルは、キー列にその綴りを出し、保存時もその綴りで書く（ドラフトの `uses_legacy_delay_key`。02 §3.2）

### 4.5 表の編集セル・削除追加時のコメントの扱い

- B1 の `table()`（罫線・反転見出し・縞・セル余白）を維持し、セルの中身をラベルから `TextEdit` に替えられるようにする。列幅・折り返しの規則（コメント列は `wrap`）は現行のまま。編集セルは等幅フォント（入力がキー記法・exe 名のため）
- 行の ✕（削除）→ ドラフトから該当行を除く。**保存時**に `toml_edit` がその項目と密着コメントを消す（02 §4.2）。画面上、削除は即時（ドラフト反映）だがファイルは保存まで不変
- ＋追加 → 末尾に空行。保存時にテーブル／配列末尾へ挿入

### 4.6 ドラフトと画面の関係（再掲・02 §3）

編集モード中に表示するのは**ドラフト**。裏で `REMAP_TABLE` が差し替わっても画面は動じない（フックは無関係）。記憶マクロ表示（§4.4）だけはライブの実行時状態を読む。

## 6. キー記法の入力・検証・入力補助

### 6.1 妥当なとき（解釈のプレビュー）

入力欄の**直下**に、解釈を人間語で薄字表示。

```
 入力  [ A-x h                    ]
         ⓘ Alt+x に続けて h（2 ストローク）
```

例: `C-h`→「Ctrl + h」、`A-x h`→「Alt+x に続けて h（2 ストローク）」、出力の `C-Right, C-Left`→「C-Right → C-Left（マクロ）」。

### 6.2 不当なとき（理由と候補）

欄の枠を警告色にし、直下に理由＋候補（あれば 1 つ）。

```
 入力  [ Bak                      ]   ← 枠は警告色
         ⚠ 未知のキー名 "Bak"。"Back" のことですか？
```

- 候補は既知キー名との近さ（編集距離）で最も近い 1 つ。無ければ理由のみ
- この状態でも入力は続けられる（打鍵の途中で一時的に不当になるのは正常）。保存は §6.4 で止まる

### 6.3 「今の前面アプリから取得」（3 秒カウントダウン）

ボタン押下後、その場でカウントダウン表示に変わり、対象アプリへ切り替える猶予を与える。

```
 押す前:  [ ⊕ 今の前面アプリから取得 ]

 押した後: [ ⏳ 3… 対象アプリを前面にしてください ]  → 2… → 1…
           → 取得: window::with_foreground_exe で前面 exe 名を application に追加
           → ボタンは元の表示へ戻る
```

- カウントダウン中はボタンを再押下不可。取得後、重複する exe 名なら足さない
- 取得元は既存の安全な公開関数（GUI に unsafe を持ち込まない。02 §6.1）

### 6.4 検証エラーのフッターバー（保存を押したとき）

保存前検証（`config::parse_str`）が落ちたら、`config-footer` に件数と先頭 1 件。クリックで該当欄へスクロール（v0.2 §5、02 §4.3）。

```
┌ [config-footer]（警告色） ────────────────────────────────────────────────┐
│ ⚠ 2 件の問題があります: 12 行目 未知のキー名 "Bak"（"Back"?）  ［次へ ▸］    │
└───────────────────────────────────────────────────────────────────────────┘
```

- ［次へ▸］で 2 件目へ。各件クリックで対応する詳細欄までスクロール＆ハイライト（逆写像の粒度は 02 §9 未決 1。当面は「先頭の該当欄まで」で可）
- 検証を通れば保存実行→フッターは消える

### 6.5 キー名一覧（`?`）

入力欄脇の `?` を押すと、使用可能キー名の一覧を小ポップアップ（`egui::popup`／子ウィンドウ）で表示。`keymap` のキー名テーブルから生成（表示専用）。

```
 入力  [ C-              ] [?]
                          └─▶ ┌ 使えるキー名 ─────────────┐
                              │ 修飾: C- A- S- W-          │
                              │ 文字: a-z 0-9              │
                              │ 特殊: Back Enter Esc Tab   │
                              │       Space Delete Home …  │
                              │ ファンクション: F1-F24     │
                              │ 方向: Left Right Up Down    │
                              └────────────────────────────┘
```

## 7. 外部変更・保存失敗・未保存クローズ（すべてウィンドウ内で）

OS モーダルは使わない。`config-footer` へ出す。外部変更の**予兆**はアドレスバーの ●（§2.2）が編集中も見えている。

### 7.1 外部変更の検出（保存直前）

開いた時点の `(mtime, size)` と保存直前が食い違うとき、保存を止めて選択を促す。

```
┌ [config-footer]（注意色） ────────────────────────────────────────────────┐
│ ⚠ config.toml が WinRemap の外で変更されています。                          │
│    ［上書き保存］  ［読み直す（編集を破棄）］  ［キャンセル］                │
└───────────────────────────────────────────────────────────────────────────┘
```

- ［上書き保存］= ドラフトを保存（外部変更は失われる）。［読み直す］= ドラフト破棄して再読込＝編集モードを作り直す。［キャンセル］= 何もしない（編集継続）
- 黙って上書きしない（02 §6.2）

### 7.2 保存失敗（権限・ディスク）

```
┌ [config-footer]（警告色） ────────────────────────────────────────────────┐
│ ⚠ 保存できませんでした: アクセスが拒否されました（config.toml）。           │
└───────────────────────────────────────────────────────────────────────────┘
```

編集モードは維持（内容は失われない）。ダイアログにしない（GUI が前面にある。v0.2 §5）。同文をステータスバーのメッセージにも残す（§5）。

### 7.3 編集に入れない／未保存で閉じる

- ［編集］押下でファイルが読めない: モード遷移せず、フッターに「編集を開始できません: <理由>」
- 未保存のまま窓を閉じる（×＝隠す。ADR 0032）: フッターに確認帯を出す（§11 未決）。既定案は「保存せずに閉じますか? ［閉じる］［キャンセル］」。閉じるとドラフト破棄

## 10. レイアウト起因の機能追加（実装前に確定が要るもの）

| 1 | **設定ファイルの切り替え**（§2.3） | 現状パスは起動時に 1 回決まる（`-c` または既定）。`gui::config_path()` は `Mutex<PathBuf>` で可変だが、**`tray` が自前の `config_path` コピーを持つ**ため、GUI で切り替えてもトレイの再読み込みは旧パスを読む | パスの持ち方を一本化（共有状態に寄せる）。切替先が読めない・検証エラーの場合の挙動（切り替えて失敗を表示 or 切り替えない）も決める。**設計判断として ADR（次の空き番号 0050 以降）を 1 件起こす** |
