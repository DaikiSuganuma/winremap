## 実施上の前提（重要）

**確認は WinRemap を 1 つだけ動かした状態で行う。** 既にインストール版が常駐している場合は先に終了させること。二重フックは挙動不定になり、結果が信用できない（ブリーフ §9・README Limitations）。

本チェックで使う設定（`[macro]` 以外は任意の既存設定でよい）:

```toml
[macro]
delay_ms = 8
record_start = "S-F10"
record_play  = "F10"
```

## Phase A — 設定の読み込みと検証（ADR 0043）

| # | 手順 | 期待する結果 | 結果 |
|---|---|---|---|
| M-1 | 上記の `[macro]` を書いて起動 | 起動する。設定ウィンドウの `[macro]` に「記憶開始 `S-F10`」「記憶終了 `S-F10`」「再生 `F10`」が並ぶ | |
| M-2 | `record_start` だけ書いて起動 | 行番号つきエラーで拒否される（`record_start` と `record_play` の両方が必要） | |
| M-3 | `record_play = "S-F10"` と、記憶キーと同じキーを書く | 行番号つきエラーで拒否される | |
| M-4 | `record_start = "LCtrl"` のように修飾キー単体を書く | 行番号つきエラーで拒否される | |
| M-5 | `[keymap.remap]` に `"S-F10" = "Home"` を書く | 行番号つきエラーで拒否され、どのキーマップと衝突したかが分かる | |
| M-6 | `[keymap.remap]` に `"F10" = "Home"`（素キー規則）を書く | `record_start`・`record_play` の両方について拒否される（素キー規則は修飾を無視して一致するため） | |
| M-7 | `[macro]` から 3 項目とも消して起動 | 起動する。記憶キーを押しても何も起きず、設定ウィンドウにも記憶の行が出ない | |
